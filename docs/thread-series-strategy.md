# Thread Series 戦略ドキュメント（Phase 6.2）

## 1. なぜ thread_series か

単発投稿に比べ、スレッド形式には以下の利点がある:

| 指標 | 単発投稿 | スレッド |
|------|---------|---------|
| 1投稿あたりの情報量 | 少 | 多 |
| 読者のエンゲージメント | 低め | 高め（継続読了） |
| CTA への誘導 | 弱 | 強（段階的説得） |
| 保存・シェア率 | 普通 | 高い（まとめ価値） |

---

## 2. thread_series の構造

```
[hook]            ← 1投稿目。読者の興味を引く。保存・リツイート誘発。
  └─ [context]    ← 背景・問題提起
       └─ [reason] ← 根拠・理由
            └─ [example] ← 具体例・事例
                 └─ [checklist] ← まとめ・チェックリスト
                      └─ [cta] ← 行動促進（LINE等）
```

デフォルトは 4 投稿（hook/context/reason/cta）。最大 8 投稿。

## 4. 各 role のライティング指針

### hook
- 1〜2行
- 問いかけ形式 or 意外性のある事実提示
- 続きを読みたくなる引っかかりを作る

### context
- 「なぜこれを伝えるのか」の背景
- 自分の経験・実績を絡める
- 読者との共感ポイントを置く

### reason
- 「なぜそうなのか」の根拠
- 数字・事例・比較を使う
- 断言できることは断言する

### example
- 具体的な実例（架空でもリアルっぽく）
- before/after 形式が効果的
- 読者が自分に置き換えられる内容

### checklist
- 箇条書き 3〜5 項目
- 読者が実行できる粒度
- 保存価値を高める

### cta
- 1〜2行
- LINEへの誘導（プレッシャーをかけない）
- 「相談はLINEで↓」など

---

## 5. 投稿間の「ため」

各投稿間に適切な「ため」を作る:

- 1〜3 投稿目：共感・問題認識の段階
- 4〜6 投稿目：解決策・根拠の提示
- 最終投稿：CTA（押しつけでなく自然な流れ）

---

## 6. 安全ガード

- 全投稿は `WAITING_REVIEW` で生成
- `draft_only` アカウントは `READY` 化不可
- 禁止キーワード・テーマは自動チェック
- 文字数超過は `review_thread_series.py` で検出
